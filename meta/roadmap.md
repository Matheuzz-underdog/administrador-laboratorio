## Fase 1: Enrutamiento, clases y HTTP

En base a lo conversado con el cliente, se requiere que en esta primera fase, usted demuestre el funcionamiento de 8 (al menos 2 de cada método HTTP: `GET`, `POST`, `PUT`, `DELETE`) de los endpoints (o rutas) que el sistema debe ser capaz de procesar.
El sistema debe:
- **Gestionar:** Crear, Consultar, Editar y Eliminar
- **Entidad:** Objeto o ser vivo que se puede definir con atributos (en esta primera fase, `pacientes`).

El sistema debe ser capaz de: 
1. Ingresar datos a todas las entidades definidas para el funcionamiento completo de la aplicación (sistema).
2. Relacionar de forma adecuada las entidades.
3. Mostrar elementos de una determinada entidad por su id.
4. Mostrar los últimos 5 elementos de algún grupo de una entidad según un criterio de ordenamiento específico.
5. Mostrar los elementos de una entidad en un rango de fecha.
6. Eliminar elementos de una entidad.
7. Eliminar la relación entre elementos de dos o más entidades.
8. Modificar datos de una entidad.
